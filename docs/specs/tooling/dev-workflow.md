# Dev Workflow: Managed Dev Session

## Purpose

Provides an `amigo dev` command that orchestrates the full development loop: watch Rust source files, snapshot engine state, incremental rebuild, restart with state restore. Accepts that Rust requires recompilation but makes the cycle as fast and invisible as possible (1-3 seconds). Build failures keep the engine running with old code.

Complements the existing asset hot-reload (instant, no restart) with a code hot-reload cycle that preserves game state across restarts.

## Public API

### CLI

```
amigo dev [--port <PORT>]
```

Implies `--api`. Starts the engine, watches `src/**/*.rs` and `Cargo.toml` for changes, and orchestrates the rebuild-restart-restore loop.

### API Methods (JSON-RPC)

```jsonc
// Save current dev state to disk
{"method": "dev.save_snapshot", "params": {}}
// -> {"result": {"path": ".amigo_dev/snapshot.ron"}}

// Query snapshot status
{"method": "dev.snapshot_status", "params": {}}
// -> {"result": {"has_snapshot": true, "scene": "gameplay", "tick": 4200}}
```

### Engine Flag

```bash
# Internal: used by amigo dev after rebuild
amigo run --api --restore-snapshot .amigo_dev/snapshot.ron
```

### MCP Tools

```
amigo_dev_rebuild()           # Trigger manual rebuild
amigo_dev_status()            # Build status: idle, building, error
amigo_dev_snapshot()          # Force snapshot without rebuild
```

## Behavior

### The Dev Loop

```
User saves .rs file
       |
       v
[amigo dev] detects change (notify watcher)
       |
       v
Send "dev.save_snapshot" via JSON-RPC (TCP:9999)
       |
       v
Engine serializes: scene ID, camera, tick, entity state
Writes to .amigo_dev/snapshot.ron
       |
       v
Engine responds with OK
       |
       v
[amigo dev] sends "engine.quit" via JSON-RPC
       |
       v
[amigo dev] runs: cargo build
       |
       +-- SUCCESS --> relaunch with --restore-snapshot
       |                    |
       |                    v
       |               Engine loads snapshot.ron
       |               Skips splash, jumps to scene
       |               Restores camera + tick + state
       |               Reconnects API server on same port
       |                    |
       |                    v
       |               "Ready!" (terminal output)
       |
       +-- FAILURE --> print compiler errors to terminal
                       engine keeps running (old code)
                       "Build failed. Fix errors and save again."
```

### Snapshot Contents

```rust
#[derive(Serialize, Deserialize)]
struct DevSnapshot {
    /// Current scene identifier
    scene_id: String,
    /// Game tick at time of snapshot
    tick: u64,
    /// Camera position and zoom
    camera: CameraSnapshot,
    /// Game clock (elapsed time)
    elapsed_secs: f64,
    /// Custom game state (opt-in via #[derive(Serialize)])
    game_state: Option<Vec<u8>>,
    /// Audio state (current music track, position)
    audio_state: Option<AudioSnapshot>,
}

#[derive(Serialize, Deserialize)]
struct CameraSnapshot {
    x: f32,
    y: f32,
    zoom: f32,
}

#[derive(Serialize, Deserialize)]
struct AudioSnapshot {
    music_track: Option<String>,
    music_position_secs: f64,
    music_volume: f32,
}
```

### Watch Mode Details

- Watches `src/**/*.rs` and `Cargo.toml` using `notify` crate (same as asset hot-reload)
- Debounce: 300ms after last file change before triggering build
- Ignores `target/` directory
- Only one build at a time; if files change during build, queues a rebuild after completion
- Terminal output: colored status messages

```
[amigo dev] Watching src/ for changes...
[amigo dev] Change detected: src/game.rs
[amigo dev] Saving snapshot...
[amigo dev] Building... (cargo build)
[amigo dev] Build OK (1.8s). Restarting...
[amigo dev] Engine ready. Snapshot restored (scene: gameplay, tick: 4200)
```

### Build Optimization

The project structure generated by `amigo new` is optimized for fast incremental builds:

- User's game crate is small (only game logic)
- Engine crates are pre-compiled and cached as git dependency
- `[profile.dev]` uses `opt-level = 1` (already in generated Cargo.toml)
- `[profile.dev.package."*"]` uses `opt-level = 2` (dependencies optimized)
- Recommend `mold` linker for Linux, `lld` for Windows

Typical incremental rebuild for game-only changes: 1-3 seconds.

### State Restore in Engine

On startup with `--restore-snapshot`:

1. Load `DevSnapshot` from the specified path
2. Skip splash screen (`SplashConfig.enabled = false` temporarily)
3. Initialize engine normally
4. After `Game::init()`, apply snapshot:
   - Set scene to `snapshot.scene_id`
   - Set camera position and zoom
   - Set game tick and elapsed time
   - Resume music track at saved position (if any)
   - Call `Game::on_dev_restore(ctx, &snapshot)` if implemented (opt-in)
5. Delete snapshot file after successful restore

### Game Trait Extension (opt-in)

```rust
pub trait Game {
    fn init(&mut self, ctx: &mut GameContext) {}
    fn update(&mut self, ctx: &mut GameContext) -> SceneAction;
    fn draw(&self, ctx: &mut DrawContext);

    /// Called after dev snapshot restore. Override to restore custom state.
    /// Default implementation does nothing.
    fn on_dev_restore(&mut self, _ctx: &mut GameContext, _snapshot: &DevSnapshot) {}

    /// Called before dev snapshot save. Return custom state to persist.
    /// Default implementation returns None.
    fn on_dev_snapshot(&self, _ctx: &GameContext) -> Option<Vec<u8>> { None }
}
```

### MCP Reconnection

When the engine restarts, the MCP server (`amigo_mcp`) must reconnect:

- `amigo_mcp` TCP connection to engine drops on engine quit
- `amigo_mcp` retries connection with exponential backoff (100ms, 200ms, 400ms, 800ms, 1.6s)
- Max retry time: 10 seconds (covers the build + restart window)
- MCP stdio connection to Claude Code stays alive throughout (only TCP reconnects)

### VS Code Integration

Ship a `.vscode/tasks.json` template with `amigo new`:

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Amigo: Dev",
            "type": "shell",
            "command": "amigo dev",
            "group": { "kind": "build", "isDefault": true },

            "isBackground": true
```

`Ctrl+Shift+B` starts the dev session. Rust compiler errors appear in VS Code's Problems panel via the `$rustc` problem matcher.

## Internal Design

### CLI Implementation (`tools/amigo_cli/src/main.rs`)

`cmd_dev()` function:

1. Verify `amigo.toml` exists
2. Start engine process: `cargo run --features api -- --api`
3. Wait for API server to be ready (poll TCP port)
4. Start `notify` file watcher on `src/` and `Cargo.toml`
5. On file change (debounced 300ms):
   a. Send `dev.save_snapshot` via JSON-RPC
   b. Send `engine.quit` via JSON-RPC
   c. Wait for process exit
   d. Run `cargo build --features api`
   e. On success: relaunch with `--restore-snapshot`
   f. On failure: print errors, wait for next file change

### API Implementation (`crates/amigo_api/src/handler.rs`)

New handler functions following existing `queue_cmd` pattern:

```rust
fn handle_dev_save_snapshot(req: &RpcRequest, state: &SharedState) -> RpcResponse {
    // Queue ApiCommand::DevSaveSnapshot
    // Engine processes in main loop, writes .amigo_dev/snapshot.ron
}

fn handle_dev_snapshot_status(req: &RpcRequest, state: &SharedState) -> RpcResponse {
    // Read .amigo_dev/snapshot.ron metadata without loading full state
}
```

### Engine Integration (`crates/amigo_engine/src/engine.rs`)

In `EngineApp::resumed()` or equivalent startup path:

```rust
if let Some(snapshot_path) = env::var("AMIGO_RESTORE_SNAPSHOT").ok() {
    let snapshot = DevSnapshot::load(&snapshot_path)?;
    // Apply to GameContext after Game::init()
    // Delete snapshot file
}
```

In the main loop, handle `ApiCommand::DevSaveSnapshot`:

```rust
ApiCommand::DevSaveSnapshot => {
    let snapshot = DevSnapshot {
        scene_id: current_scene_id.clone(),
        tick: game_context.time.tick,
        camera: CameraSnapshot::from(&game_context.camera),
        elapsed_secs: game_context.time.elapsed_secs,
        game_state: game.on_dev_snapshot(&game_context),
        audio_state: game_context.audio.snapshot(),
    };
    snapshot.save(".amigo_dev/snapshot.ron")?;
}
```

## Non-Goals

- Hot-reloading Rust code without restart (dylib loading)
- Scripting language integration
- Full entity state serialization (only opt-in via Game trait)
- Production use of snapshots (dev mode only)
- Editing code from within the engine

## Open Questions

- Whether to support `amigo dev --release` for profiling with optimized builds
- Whether snapshot format should be versioned for forward compatibility
- Maximum snapshot file size before warning
- Whether to auto-detect `mold`/`lld` and suggest installation if not found

### Behavior

#### Dev Loop — Normal Flow
- [ ] `amigo dev` starts the engine process via `cargo run --features api -- --api`
- [ ] `amigo dev` waits for the API server TCP port to be ready before proceeding
- [ ] File watcher monitors `src/**/*.rs` and `Cargo.toml` using `notify` crate
- [ ] File watcher debounces changes with a 300ms delay after the last file change
- [ ] File watcher ignores the `target/` directory
- [ ] On file change: sends `dev.save_snapshot` via JSON-RPC to running engine
- [ ] Engine serializes scene ID, camera, tick, and entity state to `.amigo_dev/snapshot.ron`
- [ ] After snapshot: sends `engine.quit` via JSON-RPC
- [ ] After engine quits: runs `cargo build --features api`
- [ ] On build success: relaunches engine with `--restore-snapshot .amigo_dev/snapshot.ron`
- [ ] On build failure: prints compiler errors to terminal, engine keeps running with old code
- [ ] On build failure: prints "Build failed. Fix errors and save again."
- [ ] Only one build runs at a time; if files change during build, a rebuild is queued after completion
